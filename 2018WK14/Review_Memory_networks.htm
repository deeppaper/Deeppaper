리뷰 - Memory Networks

Skip to end of metadata
  • Created by 임 도형, last modified on Apr 09, 2018
Go to start of metadata

개요

입력 인코딩(I), 메모리 업데이트(G), 출력 찾기(O), 출력 인코딩(R)로 구조로 구성된 시스템이 제안.

실험으로는 일반 영문을 사용하여, 질문에 대한 답변을 하였다.


이러한 구조를 가지고 일반적인 문제를 해결할 수 있음을 보임.


이전 Neural Turing Machin에 이은 논문이며, 

이후 같은 저자가 이를 보완한 하지만 완전 다른 구조의 End-To-End Memory Networks를 제안함.

하지만  DeepMind의 Differential Neural Computers에 눌렸다. 


논문 의미

아주 간단한 구조의, 하지만 메모리를 사용하는 구조로 일반 문제를 해결할 수 있다.

그 간단한 Turing Machine이 일반 문제를 해결할 수 있다는 것에 버금갈 정도의 강한 임팩트이다.


시스템 구조


전체 시스템은 I, G, O, R의 네트웤으로 구성

  • I : feature = I(x). 입력을 인코딩
  • G : mi = G(mi, I(x), m). 메모리 번지를 업데이트
  • O : o = O(I(x), m). 입력과 메모리를 가지고 출력을 찾아냄
  • R : R(o) -> r. 출력을 디코딩


각 네트웤을 학습시킬 수 있다.

각 네트웤은 NN이 아니어도 된다.


실험 데이타

일반 영문.

?
Joe went to the kitchen.
Fred went to the kitchen.
Joe picked up the milk.
Joe travelled to the office.
Joe left the milk.
Joe went to the bathroom.
Where is the milk now? A: office
Where is Joe? A: bathroom
Where was Joe before the office? A: kitchen


답변이 앞의 문장 하나로 구할 수 없다.

몇개의 문장을 기억(memory, 저장소)에서 구해야 한다.



학습을 위한 loss function


S( )는 적합성. So( )는 O의 출력에 대한. Sr( )은 디코딩된것에 대한. 

1개의 입력 x에 대하여 2개의 메모리 번지의 내용 mo1, mo2를 찾는다.

mo1, mo2는 선택된 메모리의 내용.  So(x,mo1)은 입력 x에 대하여 선태된 mo1의 적합성.


항목 중 bar가 붙은 것은 오정답인 경우. 

첫 항목은 입력(x)과 찾아낸 기억 내용(mo1)과의 적합성

두번째 항목은 이전 결과([x, mo1])과 찾아낸 두번째 기억 내용( mo2)과의 적합성

서번째 항목은 찾아낸 결과([x, mo1, mo2])와 인코딩된 출력(r)과 의 적합성


요약하면 입력(x)이 들어왔을 때 출력(r)을 내기위한 기억 2개(mo1, mo2)를 찾는다. 그리고 이를 가지고 출력을 찾고, 이를 잘 디코딩하는가이다.
